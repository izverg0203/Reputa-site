# 06-animation-guidelines.md  
## Motion & Interaction Guidelines — Анимации и динамика Reputa

---

# 1. Назначение документа

Цель этого документа — описать **правила анимаций, переходов, микро-взаимодействий и поведения интерфейсов** продукта Reputa.

Reputa — это сайт и платформа с динамикой уровня Apple.  
Здесь motion — не украшение, а часть бренда:

- передаёт интеллект AI,  
- делает продукт премиальным,  
- помогает понимать данные,  
- создаёт ощущение “живого интерфейса”.

---

# 2. Основные принципы анимаций

### 2.1 Мягкость  
— никакой резкости, рывков, дерганий.  
— easing должен быть плавным и доверительным.

### 2.2 Умность  
— анимация должна помогать понять структуру UI.  
— движения подсказывают: “что важно → куда смотреть”.

### 2.3 Живость  
— микропульсации, мягкие свечения, дыхание элементов.  
— интерфейс ощущается как организм, а не набор блоков.

### 2.4 Технологичность  
— световые акценты, эффекты глубокого пространства, параллакс.  
— всё в рамках «тёплой технологичности».

---

# 3. Технические параметры motion

### 3.1 Длительность (timing)
- компоненты: **200–280 ms**
- секции: **360–480 ms**
- большие переходы: **500–650 ms**

### 3.2 Easing
Рекомендуемые:
- `ease-out`
- `ease-in-out`
- `cubic-bezier(0.16, 1, 0.3, 1)` ← фирменная Apple-like кривая
- для пружины: `spring` с низкой жёсткостью

### 3.3 Инструменты
- **Framer Motion**
- **GSAP (по необходимости)**
- **Lottie** (для органических AI-эффектов)

---

# 4. Анимации по компонентам

## 4.1 Hero-секция
- параллакс заднего фона (waves/particles)  
- плавное появление заголовка  
- фейд-ин UI-превью (с левого нижнего угла → центр)  
- элементы данных оживают поочерёдно:  
  - теги sentiment  
  - график  
  - уведомление “1 критический отзыв”

## 4.2 Карточки (Cards)
- при появлении: fade + небольшое смещение  
- hover: увеличение 1–3%, тень +15%  
- AI-инсайты пульсируют тонким свечением  
- critical-отзывы слегка вибрируют (микро-внимание)

## 4.3 Кнопки
- hover: мягкое расширение (scale 1.02)  
- click: лёгкая пружина  
- CTA — свечение по краям при наведении  

## 4.4 AI-блок (тёмная секция)
- точки данных перемещаются по синусоиде  
- диаграмма «рисуется» на глазах  
- линии графиков имеют “elastic draw” эффект  
- слабые зоны загораются мягким красным акцентом

## 4.5 Фид отзывов
- отзывы подгружаются сверху вниз с инерцией  
- теги sentiment вспыхивают коротким свето-акцентом  
- карточки критических отзывов «пульсируют» каждые 10 сек (очень мягко)

## 4.6 AI-помощник (чат)
Важный элемент бренда.

### Закрытый вид:
- плавающее “дыхание” (scale 0.98 → 1.02)  
- свечения вокруг кнопки  
- при наведении — лёгкий tilt

### Открытый чат:
- scale 0.95 → 1.0 (elastic)  
- сообщения появляются с fade + slight left slide  
- курсор «AI думает» — три мягкие движущиеся точки

# 5. Анимации по секциям сайта

## 5.1 “Как работает AI”
Каждый шаг появляется при скролле:
- карточка 1 → карточка 2 → карточка 3  
- стрелки или связь между ними слегка пульсируют  

## 5.2 AI-Аналитика (тёмная секция)
- donut chart появляется сегментами  
- топ-проблем появляются с прозрачности 0 → 1  
- подсветки категорий — короткие волны света  

## 5.3 Блок отзывов
- автоматический слайдер  
- эффекты инерции при перелистывании  
- теги sentiment вспыхивают лёгким halo  

## 5.4 Тарифные планы
- карточки слегка наклоняются при hover  
- активный тариф подсвечивается мягким светом  

---

# 6. Переходы между секциями (scroll storytelling)

Сайт Reputa должен работать как **mini-storytelling**, где каждая секция — как кадр в фильме.

### Переходы:
- параллакс между слоями  
- изменение фона (светлый → тёмный → светлый)  
- крупные элементы смещаются на 10–20px  
- графики постепенно “появляются” при скролле  

### Общий эффект:
Пользователь чувствует, что двигается через “AI-экосистему”, а не статичный сайт.

---

# 7. **Frontend Design Skill — критически важная роль в проекте**

Чтобы сайт Reputa выглядел так, как мы планируем — динамика уровня Apple + тёплая технологичность + органические AI-эффекты — нам нужен не просто фронтенд.

Нужен **Frontend Designer** (или AI-агент с такими инструкциями).

---

## 7.1 Что делает Frontend Designer?

Это специалист, который сочетает:

### ✔ Motion design  
### ✔ Анимации интерфейсов  
### ✔ Композицию и визуальную ритмику  
### ✔ Реализацию эффектов в коде  
### ✔ Работа с GSAP, Framer Motion  
### ✔ Чувство «премиальности» интерфейса  

Это не “программист”, и не “дизайнер”.

Это кто-то между ними.

---

## 7.2 Почему он нужен Reputa

Потому что продукт строится на ощущении:

- премиального уровня  
- умного AI  
- живой платформы  
- медицинской технологичности  

Без грамотного Frontend Design — сайт будет выглядеть:

❌ как обычная SaaS-страница  
❌ без динамики  
❌ без ощущения “интеллектуальности”  
❌ без эмоциональной глубины  

Это критически важно, чтобы клиники восприняли Reputa как **сильный, современный, инновационный продукт**.

---

## 7.3 Как Frontend Design Skill интегрируется в наш pipeline

В проекте Reputa эта компетенция будет:

- в требованиях к анимациям (этот документ)  
- в Design Guidelines  
- в UI Kit  
- в Site Structure  
- в PRD  

А значит:

### любой AI-генератор (Cursor, Gemini, Lovable)  
сможет работать *как Frontend Designer*,  
потому что у него есть полный набор инструкций.

---

# 8. Запреты и анти-паттерны

Нельзя:
- использовать резкие анимации  
- делать “прыгающие” элементы  
- перегружать графику  
- ставить анимации на формы ввода  
- делать бездумные появления  
- использовать устаревшие JS-анимации  

---

# 9. Итог

Анимация — часть ДНК бренда Reputa.  
Она должна быть:

- мягкой  
- интеллектуальной  
- премиальной  
- тёплой  
- живой  

Этот документ — основа для построения motion-системы и Frontend Design поведения для всех разработчиков и AI-инструментов.
