# 02-masterplan.md — Masterplan развития Reputa  
## Дорожная карта продукта, сайта и AI-экосистемы

---

# 1. Видение (Vision)

Reputa становится **основной AI-платформой репутационного управления для клиник**.  
Она заменяет ручной мониторинг, рутинные ответы и поверхностные отчёты — создаёт *глубокий AI-уровень репутационного интеллекта*.

Ценности:
- Прозрачность репутации  
- Предупреждение кризисов  
- Рост доверия и заявок  
- Умная автоматизация  

Эмоция бренда:  
**«Тёплая технологичность, которая защищает клинику и ускоряет рост»**

---

# 2. Общая структура продукта

Reputa состоит из трёх слоёв:

### Слой 1: AI-Мониторинг  
- сбор отзывов из всех площадок  
- единый фид  
- тревоги по критическим отзывам  

### Слой 2: AI-Интеллект  
- тональность  
- категории проблем  
- динамика и слабые зоны  
- сравнение с конкурентами  
- выявление «токсичных» зон  

### Слой 3: AI-Реакция  
- генерация ответов  
- согласование и отправка  
- шаблоны  
- «речевые скрипты» администраторам  

---

# 3. Дорожная карта (Roadmap)

## Фаза 1 — MVP (1–2 месяца)
Задача: получить 5–10 платящих клиентов.

**Включает:**
- Лендос с AI-аудитом  
- Функции мониторинга (Яндекс, 2GIS, Google, VK)  
- Тональность и категории  
- Ответы с AI-черновиками  
- Кабинет с 3 главными метриками  

**Главный результат:**  
Клиники чувствуют облегчение → быстро видят ценность → сохраняются.

---

## Фаза 2 — Growth (2–4 месяца)
Цель: ликвидное отличие от конкурентов.

**Новые функции:**
- сравнение филиалов  
- рейтинг врачей и подразделений  
- прогноз репутации  
- автоматические рекомендации  
- обучение администраторов (AI-скрипт-тренер)  

**Коммерция:**
- тарифы для сетей  
- мультиаккаунты  
- отчёты для собственников  

---

## Фаза 3 — AI-Clinic OS (6–12 месяцев)
Reputa становится «операционной системой репутации клиники».

**Будущие функции:**
- автоматизация ответа во всех соцсетях  
- AI-ассистент для пациентов  
- анализ звонков и переписок (через интеграции)  
- KPI-панель для главврача  
- репутационный «Early Warning System»  
- Smart-push: «через 2 недели рейтинг упадёт → ожидание на ресепшене»

# 4. Архитектура разработки

## Frontend
- Next.js + React  
- Tailwind + shadcn/ui  
- Анимации: Framer Motion, Lottie  

## Backend / Data
- Supabase (DB + auth + storage)  
- n8n (интеграции и webhooks для ответов)  
- OpenAI/Gemini (аналитика и ответы)  

## Интеграции
- Yandex Maps  
- 2GIS  
- Google Reviews  
- VK  

---

# 5. Стратегия сайта

Сайт выполняет три функции:

### 1. Продажи  
- AI-аудит → главный лидогенератор  
- демо → тех, кто хочет понять продукт  

### 2. Объяснение ценности  
Каждый блок сайта = усиление эмоции «AI защищает и ускоряет».

### 3. Визуальная демонстрация технологии  
- интерактивы  
- тёмный AI-блок  
- динамические графики  
- анимации по скроллу  

---

# 6. Стратегия AI-помощника

AI-помощник — важный элемент сайта:

### Он должен:
- объяснить, что делает Reputa  
- подсказать тариф  
- провести мини-аудит  
- собрать лиды  
- вовлечь, а не раздражать  

### Тональность:
- мягкая  
- экспертная  
- уверенная  

---

# 7. KPI успеха продукта

### Для сайта:
- конверсия AI-аудита 8–15%  
- заявки на демо 5–10%  
- удержание на странице > 1:40  

### Для платформы:
- сокращение времени ответа 30–50%  
- рост позитивных отзывов 10–25%  
- снижение критических отзывов 20–40%  
- рост заявок у клиента 10–20%  

---

# 8. Риски
- зависимость от API площадок  
- качество AI-черновиков  
- нагрузка на n8n  
- непонимание клиник, что такое «AI-анализ»  

---

# 9. Принципы команды
- быстрые спринты  
- фокус на реальную пользу клинике  
- тестирование на реальных отзывах  
- минимальный шум, максимальная ясность
